## 值类型

`VALUE_TYPE_NAMES` 定义了 WASM 支持的值类型：

| 字节码 | 类型名 | 说明 |
|--------|--------|------|
| `0x7F` | `i32` | 32 位整数 |
| `0x7E` | `i64` | 64 位整数 |
| `0x7D` | `f32` | 32 位浮点数 |
| `0x7C` | `f64` | 64 位浮点数 |
| `0x7B` | `v128` | 128 位向量 (SIMD) |
| `0x70` | `funcref` | 函数引用 |
| `0x6F` | `externref` | 外部引用 |
| `0x40` | `void` | 无返回值 |

## 控制流指令

控制流指令用于管理程序执行流程。

### 基本控制

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x00` | `unreachable` | 无 | `[] → trap` | 触发陷阱，表示不可达代码 |
| `0x01` | `nop` | 无 | `[] → []` | 无操作 |
| `0x0F` | `return` | 无 | `[t*] → []` | 从当前函数返回 |

### 块结构

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x02` | `block` | `blocktype` | `[] → []` | 开始一个块，blocktype 指定结果类型 |
| `0x03` | `loop` | `blocktype` | `[] → []` | 开始一个循环块 |
| `0x04` | `if` | `blocktype` | `[i32] → []` | 条件块，消耗栈顶 i32 作为条件 |
| `0x05` | `else` | 无 | `[] → []` | if 的 else 分支 |
| `0x0B` | `end` | 无 | `[] → []` | 结束 block/loop/if 结构 |

**blocktype 参数格式：**
- `0x40` - 无返回值
- `0x7F` - 返回 i32
- `0x7E` - 返回 i64
- `0x7D` - 返回 f32
- `0x7C` - 返回 f64
- LEB128 编码的类型索引 - 多返回值

### 跳转指令

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x0C` | `br` | `labelidx: u32` | `[t*] → []` | 无条件跳转到标签 |
| `0x0D` | `br_if` | `labelidx: u32` | `[t*, i32] → [t*]` | 条件为真时跳转 |
| `0x0E` | `br_table` | `vec(labelidx), labelidx` | `[t*, i32] → []` | 基于索引的表跳转 |

**参数说明：**
- `labelidx` - LEB128 编码的标签索引，0 表示最内层块
- `br_table` 参数：先是标签数组长度，然后是各标签索引，最后是默认标签

**示例：**
```wasm
;; br_if 示例：如果栈顶为非零则跳转到外层块
block
  i32.const 1
  br_if 0      ;; 跳转到当前 block 的 end
end

;; br_table 示例
i32.const 1
br_table 0 1 2  ;; 索引 0→标签0, 索引 1→标签1, 默认→标签2
```

## 调用指令

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x10` | `call` | `funcidx: u32` | `[t1*] → [t2*]` | 直接调用函数 |
| `0x11` | `call_indirect` | `typeidx: u32, tableidx: u32` | `[t1*, i32] → [t2*]` | 通过表间接调用 |
| `0x12` | `return_call` | `funcidx: u32` | `[t1*] → []` | 尾调用优化 |
| `0x13` | `return_call_indirect` | `typeidx: u32, tableidx: u32` | `[t1*, i32] → []` | 间接尾调用 |

**参数说明：**
- `funcidx` - 函数索引（LEB128 编码）
- `typeidx` - 函数类型索引
- `tableidx` - 表索引（通常为 0）
- 栈顶 i32 是表中的元素索引（仅 indirect 调用）

**示例：**
```wasm
;; 直接调用函数 3
i32.const 42
call 3

;; 间接调用：从表 0 中索引 5 位置获取函数，类型为 0
i32.const 10
i32.const 5    ;; 表索引
call_indirect 0 0
```

## 变量指令

### 栈操作

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x1A` | `drop` | 无 | `[t] → []` | 丢弃栈顶值 |
| `0x1B` | `select` | 无 | `[t, t, i32] → [t]` | 根据条件选择两个值之一 |

**select 行为：** 如果 i32 非零，返回第一个值；否则返回第二个值。

### 局部变量

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x20` | `local.get` | `localidx: u32` | `[] → [t]` | 读取局部变量 |
| `0x21` | `local.set` | `localidx: u32` | `[t] → []` | 写入局部变量 |
| `0x22` | `local.tee` | `localidx: u32` | `[t] → [t]` | 写入并保留栈顶值 |

**参数说明：**
- `localidx` - 局部变量索引，包括函数参数（参数在前，局部变量在后）

### 全局变量

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x23` | `global.get` | `globalidx: u32` | `[] → [t]` | 读取全局变量 |
| `0x24` | `global.set` | `globalidx: u32` | `[t] → []` | 写入全局变量（必须是 mutable） |

### 表操作

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x25` | `table.get` | `tableidx: u32` | `[i32] → [t]` | 读取表元素 |
| `0x26` | `table.set` | `tableidx: u32` | `[i32, t] → []` | 写入表元素 |

## 内存指令

### 内存加载

所有加载指令的参数格式：`align: u32, offset: u32`（均为 LEB128 编码）

| 操作码 | 指令 | 栈行为 | 说明 |
|--------|------|--------|------|
| `0x28` | `i32.load` | `[i32] → [i32]` | 从内存加载 32 位整数 |
| `0x29` | `i64.load` | `[i32] → [i64]` | 从内存加载 64 位整数 |
| `0x2A` | `f32.load` | `[i32] → [f32]` | 从内存加载 32 位浮点 |
| `0x2B` | `f64.load` | `[i32] → [f64]` | 从内存加载 64 位浮点 |
| `0x2C` | `i32.load8_s` | `[i32] → [i32]` | 加载 8 位并符号扩展 |
| `0x2D` | `i32.load8_u` | `[i32] → [i32]` | 加载 8 位并零扩展 |
| `0x2E` | `i32.load16_s` | `[i32] → [i32]` | 加载 16 位并符号扩展 |
| `0x2F` | `i32.load16_u` | `[i32] → [i32]` | 加载 16 位并零扩展 |
| `0x30` | `i64.load8_s` | `[i32] → [i64]` | 加载 8 位并符号扩展为 i64 |
| `0x31` | `i64.load8_u` | `[i32] → [i64]` | 加载 8 位并零扩展为 i64 |
| `0x32` | `i64.load16_s` | `[i32] → [i64]` | 加载 16 位并符号扩展为 i64 |
| `0x33` | `i64.load16_u` | `[i32] → [i64]` | 加载 16 位并零扩展为 i64 |
| `0x34` | `i64.load32_s` | `[i32] → [i64]` | 加载 32 位并符号扩展为 i64 |
| `0x35` | `i64.load32_u` | `[i32] → [i64]` | 加载 32 位并零扩展为 i64 |

**参数说明：**
- `align` - 对齐指数（2^align 字节对齐），如 align=2 表示 4 字节对齐
- `offset` - 静态偏移量，与栈顶地址相加
- 有效地址 = 栈顶 i32 + offset

**示例：**
```wasm
;; 从地址 100+8=108 加载 i32
i32.const 100
i32.load align=2 offset=8
```

### 内存存储

| 操作码 | 指令 | 栈行为 | 说明 |
|--------|------|--------|------|
| `0x36` | `i32.store` | `[i32, i32] → []` | 存储 32 位整数 |
| `0x37` | `i64.store` | `[i32, i64] → []` | 存储 64 位整数 |
| `0x38` | `f32.store` | `[i32, f32] → []` | 存储 32 位浮点 |
| `0x39` | `f64.store` | `[i32, f64] → []` | 存储 64 位浮点 |
| `0x3A` | `i32.store8` | `[i32, i32] → []` | 存储低 8 位 |
| `0x3B` | `i32.store16` | `[i32, i32] → []` | 存储低 16 位 |
| `0x3C` | `i64.store8` | `[i32, i64] → []` | 存储低 8 位 |
| `0x3D` | `i64.store16` | `[i32, i64] → []` | 存储低 16 位 |
| `0x3E` | `i64.store32` | `[i32, i64] → []` | 存储低 32 位 |

**栈顺序：** 先压入地址，再压入值。

### 内存管理

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x3F` | `memory.size` | `memidx: u32` | `[] → [i32]` | 获取当前内存页数（1页=64KB） |
| `0x40` | `memory.grow` | `memidx: u32` | `[i32] → [i32]` | 增加内存页数，返回旧页数或 -1 |

## 常量指令

| 操作码 | 指令 | 参数 | 栈行为 | 说明 |
|--------|------|------|--------|------|
| `0x41` | `i32.const` | `value: i32` | `[] → [i32]` | 压入 i32 常量（LEB128 有符号） |
| `0x42` | `i64.const` | `value: i64` | `[] → [i64]` | 压入 i64 常量（LEB128 有符号） |
| `0x43` | `f32.const` | `value: f32` | `[] → [f32]` | 压入 f32 常量（4 字节 IEEE 754） |
| `0x44` | `f64.const` | `value: f64` | `[] → [f64]` | 压入 f64 常量（8 字节 IEEE 754） |

**示例：**
```wasm
i32.const 42       ;; 编码: 0x41 0x2A
i32.const -1       ;; 编码: 0x41 0x7F (LEB128)
i64.const 100      ;; 编码: 0x42 0x64
f32.const 3.14     ;; 编码: 0x43 + 4字节浮点
```

## 附录：LEB128 编码

WASM 使用 LEB128（Little Endian Base 128）编码整数：

```
无符号 LEB128:
- 每字节低 7 位是数据，最高位表示是否继续
- 0x00-0x7F: 单字节值
- 0x80+: 多字节值

有符号 LEB128:
- 同上，但最后一个字节的第 6 位是符号位
- 正数最后字节 < 0x40
- 负数最后字节 >= 0x40
```

**示例：**
```
42    → 0x2A           (单字节)
128   → 0x80 0x01      (多字节: 128 = 0 + 128*1)
-1    → 0x7F           (有符号，高位扩展)
-128  → 0x80 0x7F      (有符号多字节)
```
